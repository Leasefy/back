# Milestone v1.3: Subscription Restructuring & Unified Evaluations

**Status:** ✅ SHIPPED 2026-04-04
**Phases:** 25-28
**Total Plans:** 13

## Overview

Reestructuracion de suscripciones (STARTER/PRO/FLEX) con modelo pay-per-evaluation, endpoint unificado que consume el microservicio de agentes, sistema de creditos, y billing FLEX por canon.

## Phases

### Phase 25: Tier Migration & Access Control

**Goal:** Los tiers de suscripcion reflejan el nuevo modelo STARTER/PRO/FLEX, con pricing actualizado y acceso al scoring restringido segun rol.
**Depends on:** Phase 24
**Plans:** 3 plans

- [x] 25-01: Rename enum FREE/PRO/BUSINESS to STARTER/PRO/FLEX (DB migration, Prisma schema, TS enum, seed data)
- [x] 25-02: Update all source code references from FREE/BUSINESS to STARTER/FLEX
- [x] 25-03: Restrict scoring endpoints to tenant-only access

### Phase 26: Agent Credits System

**Goal:** Landlords e inmobiliarias pueden comprar creditos de evaluacion por adelantado, consultar su saldo, y ver el historial de transacciones.
**Depends on:** Phase 25
**Plans:** 3 plans

- [x] 26-01: Prisma schema (AgentCredit + AgentCreditTransaction models, evaluationCreditPrice)
- [x] 26-02: AgentCreditsModule (purchase endpoint, balance query, atomic deductCredits)
- [x] 26-03: Transaction history endpoint with pagination

### Phase 27: Unified Evaluation Endpoint

**Goal:** Landlord/inmobiliaria puede solicitar una evaluacion unificada de un aplicante; el backend orquesta la llamada al microservicio de agentes, almacena el resultado, y aplica las reglas de precio y limite por tier.
**Depends on:** Phase 25, Phase 26
**Plans:** 4 plans

- [x] 27-01: Schema (EvaluationResult, EvaluationTransaction, EvaluationUsage + env config)
- [x] 27-02: AgentMicroClient (Node fetch wrapper for micro)
- [x] 27-03: EvaluationsModule (orchestration: ownership, idempotency, credits, micro call, polling)
- [x] 27-04: Pricing enforcement hardening (tier verification, PRO 30/month, error paths)

### Phase 28: FLEX Billing

**Goal:** Agencias FLEX pueden trackear el canon total que administran, el sistema aplica el split del 1% en pagos PSE, y el dashboard muestra el cobro estimado.
**Depends on:** Phase 25
**Plans:** 3 plans

- [x] 28-01: CanonTracking Prisma model + migration
- [x] 28-02: FlexBillingModule + PSE auto-tracking + manual canon endpoint
- [x] 28-03: FLEX dashboard endpoint (canon total, 1% fee, history)

## Milestone Summary

**Key Decisions:**
- ALTER TYPE RENAME VALUE for non-destructive enum migration (25-01)
- AgentCreditTransaction relates to User only, not AgentCredit (26-01)
- deductCredits uses atomic conditional updateMany (26-02)
- Node 18+ built-in fetch for micro client — no new packages (27-02)
- Client-driven polling, no background scheduler (27-03)
- FLEX tier never calls deductCredits (27-03)
- PRO race condition accepted with logger warning (27-04)
- 1% fee on rentAmount only, not totalAmount (28-02)
- PSE auto-tracking non-blocking (try/catch) (28-02)

**Issues Resolved:**
- Phase 25 verification gap (Tenant PRO pricing) resolved in Phase 26 seed update
- Phase 25 SC#6 (evaluation auth) resolved by Phase 27 implementation

**Technical Debt:**
