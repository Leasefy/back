# v1.3 Backend API — Guia de Integracion Frontend

**Version:** v1.3 — Subscription Restructuring & Unified Evaluations
**Fecha:** 2026-04-04
**Base URL:** `{API_URL}` (todas las rutas requieren `Authorization: Bearer {token}`)

---

## Resumen de Cambios

El backend v1.3 agrega 3 modulos nuevos y modifica el acceso a scoring:

1. **Tiers renombrados**: FREE → STARTER, BUSINESS → FLEX (PRO se mantiene)
2. **Agent Credits**: sistema de creditos para pagar evaluaciones
3. **Evaluaciones Unificadas**: endpoint para que landlords soliciten evaluaciones de aplicantes
4. **FLEX Billing**: tracking de canon y dashboard para agencias FLEX
5. **Scoring restringido**: los endpoints de scoring ahora son solo para tenants

---

## CAMBIO IMPORTANTE: Tiers Renombrados

| Antes (v1.0) | Ahora (v1.3) | Precio/mes |
|---------------|--------------|------------|
| `FREE` | `STARTER` | $0 COP |
| `PRO` | `PRO` (sin cambio) | $149,000 COP |
| `BUSINESS` | `FLEX` | $0 COP (billing por canon 1%) |

**Accion frontend:** Reemplazar toda referencia a `FREE` por `STARTER` y `BUSINESS` por `FLEX` en:
- UI de planes/pricing
- Logica condicional basada en tier
- Labels, badges, textos

El enum `SubscriptionPlan` ahora es `STARTER | PRO | FLEX`.

---

## CAMBIO IMPORTANTE: Scoring ahora es Tenant-Only

Los endpoints de scoring existentes ahora devuelven **403 Forbidden** para landlords y agentes:

- `GET /scoring/:applicationId` → 403 si no es TENANT
- `GET /scoring/:applicationId/explanation` → 403 si no es TENANT

**Accion frontend:** Landlords/agentes ya NO pueden llamar estos endpoints directamente. Deben usar el nuevo endpoint de evaluaciones (ver seccion abajo).

## Nuevos Endpoints

### 1. Agent Credits (Creditos de Evaluacion)

**Roles permitidos:** `LANDLORD`, `AGENT`
**Prefijo:** `/agent-credits`

#### GET /agent-credits/balance

Consulta el saldo actual de creditos del usuario.

**Response 200:**
```json
{
  "balance": 8
}
```

Si el usuario nunca compro creditos, retorna `balance: 0`.

---

#### POST /agent-credits/purchase

Compra un pack de creditos via PSE.

**Request body:**
```json
{
  "packSize": 5,
  "psePaymentData": {
    "documentType": "CC",
    "documentNumber": "1234567890",
    "bankCode": "BANCOLOMBIA",
    "holderName": "Juan Perez",
    "phoneNumber": "+573001234567"
  }
}
```

| Campo | Tipo | Requerido | Valores |
|-------|------|-----------|---------|
| `packSize` | int | si | `1`, `5`, `10`, `20` |
| `psePaymentData.documentType` | string | si | `CC`, `CE`, `NIT`, `PASAPORTE` |
| `psePaymentData.documentNumber` | string | si | 6-15 digitos |
| `psePaymentData.bankCode` | string | si | Enum `ColombianBank` (BANCOLOMBIA, DAVIVIENDA, etc.) |
| `psePaymentData.holderName` | string | si | Min 3 caracteres |
| `psePaymentData.phoneNumber` | string | no | Celular colombiano (+573...) |

**Precios por pack (COP):**

| Pack | Precio | Descuento |
|------|--------|-----------|
| 1 credito | $42,000 | — |
| 5 creditos | $189,000 | ~10% |
| 10 creditos | $350,000 | ~17% |
| 20 creditos | $630,000 | ~25% |

**Response 201:**
```json
{
  "balance": 13,
  "creditsAdded": 5,
  "payment": {
    "transactionId": "PSE-1712234567-ABC123",
    "status": "SUCCESS"
  }
}
```

**Errores:**
| Status | Causa |
|--------|-------|
| 400 | Pack invalido o pago PSE rechazado |

**Nota PSE mock:** El resultado depende del ultimo digito del `documentNumber`:
- `0` → FAILURE (fondos insuficientes)
- `1` → FAILURE (banco rechaza)
- `9` → PENDING (verificacion bancaria)
- Otros → SUCCESS

### 2. Evaluaciones Unificadas

**Roles permitidos:** `LANDLORD` (AGENT hereda acceso automaticamente)
**Prefijo:** `/evaluations`

#### POST /evaluations/:applicationId

Solicita una evaluacion unificada del aplicante. El backend valida suscripcion, cobra creditos, llama al microservicio de agentes, y retorna un `runId` para polling.

**No requiere body** — solo el `applicationId` en la URL.

**Response 202:**
```json
{
  "runId": "run_abc123",
  "status": "PENDING"
}
```

**Idempotencia:** Si ya existe una evaluacion PENDING o COMPLETED para esa aplicacion, retorna la existente SIN cobrar de nuevo. Solo si la anterior fue FAILED se permite reintentar (cobra de nuevo).

**Logica de cobro por tier:**

| Tier | Precio por evaluacion | Creditos deducidos | Limite mensual |
|------|----------------------|-------------------|----------------|
| STARTER | $42,000 COP | 1 | Sin limite |
| PRO | $21,000 COP | 1 | 30/mes |
| FLEX | $0 (gratis) | 0 | Sin limite |

**Errores:**

| Status | Causa | Mensaje |
|--------|-------|---------|
| 400 | Creditos insuficientes | "Saldo de creditos insuficiente. Compra mas creditos para continuar." |
| 403 | Sin suscripcion activa | "Se requiere una suscripcion activa para solicitar evaluaciones" |
| 403 | No es dueno de la propiedad | "No tenes acceso a esta solicitud" |
| 404 | Aplicacion no existe | Prisma NotFound |
| 429 | PRO supero 30/mes | "Limite de 30 evaluaciones mensuales alcanzado" |
| 503 | Microservicio caido | "El microservicio de agentes no esta disponible" |

---

#### GET /evaluations/:applicationId/result

Consulta el estado/resultado de una evaluacion. Si esta PENDING, el backend intenta obtener el resultado del microservicio automaticamente.

**Response 200:**

Cuando esta pendiente:
```json
{
  "runId": "run_abc123",
  "status": "PENDING",
  "createdAt": "2026-04-04T15:30:00.000Z"
}
```

Cuando completo:
```json
{
  "runId": "run_abc123",
  "status": "COMPLETED",
  "result": {
    // JSON completo del microservicio de agentes
    // Estructura variable — tratar como objeto generico
  },
  "createdAt": "2026-04-04T15:30:00.000Z"
}
```

Cuando fallo:
```json
{
  "runId": "run_abc123",
  "status": "FAILED",
  "error": "Descripcion del error",
  "createdAt": "2026-04-04T15:30:00.000Z"
}
```

**Errores:**

| Status | Causa |
|--------|-------|
| 403 | No es dueno de la propiedad |
| 404 | No existe evaluacion para esta aplicacion |

**Patron de polling recomendado:**
1. Llamar POST /evaluations/:applicationId → recibir runId
2. Esperar 2-3 segundos
3. Llamar GET /evaluations/:applicationId/result
4. Si status es PENDING, esperar 3-5 segundos y reintentar
5. Si status es COMPLETED, mostrar resultado
6. Si status es FAILED, mostrar error y ofrecer reintentar

### 3. FLEX Billing (Solo Agencias)

**Roles permitidos:** Miembros de agencia (via `AgencyMemberGuard`)
**Prefijo:** `/inmobiliaria/flex-billing`
**Header requerido:** `x-agency-id: {agencyId}` (mismo que otros endpoints de inmobiliaria)

#### POST /inmobiliaria/flex-billing/canon

Reporta manualmente el canon de un arriendo que no se pago via PSE.
**Permiso requerido:** `cobros:edit`

**Request body:**
```json
{
  "consignacionId": "uuid-de-la-consignacion",
  "canonAmount": 1500000,
  "month": "2026-04",
  "paymentReference": "Transferencia #12345"
}
```

| Campo | Tipo | Requerido | Validacion |
|-------|------|-----------|------------|
| `consignacionId` | UUID | si | Debe pertenecer a la agencia |
| `canonAmount` | int | si | > 0, en COP |
| `month` | string | si | Formato `YYYY-MM` |
| `paymentReference` | string | no | Referencia libre |

**Response 201:**
```json
{
  "id": "uuid",
  "agencyId": "uuid",
  "consignacionId": "uuid",
  "canonAmount": 1500000,
  "leasifyFee": 15000,
  "month": "2026-04",
  "source": "MANUAL",
  "paymentReference": "Transferencia #12345",
  "createdAt": "2026-04-04T15:30:00.000Z"
}
```

**Nota:** `leasifyFee` se calcula automaticamente como 1% del `canonAmount` (redondeado).

**Errores:**

| Status | Causa |
|--------|-------|
| 404 | Consignacion no encontrada en esta agencia |

**Nota:** Si ya existe un registro para esa consignacion+mes, se ACTUALIZA (upsert). No genera duplicados.

#### GET /inmobiliaria/flex-billing/dashboard

Dashboard de billing FLEX: canon total administrado y cobro estimado del 1%.
**Permiso requerido:** `dashboard:view`

**Query params:**

| Param | Tipo | Default | Ejemplo |
|-------|------|---------|---------|
| `year` | int | todos | `2026` |

**Response 200:**
```json
{
  "canonTotal": 15000000,
  "leasifyFeeTotal": 150000,
  "estimatedCharge": 150000,
  "recordCount": 10,
  "history": [
    {
      "id": "uuid",
      "consignacionId": "uuid",
      "canonAmount": 1500000,
      "leasifyFee": 15000,
      "month": "2026-04",
      "source": "PSE_AUTO",
      "pseTransactionId": "PSE-...",
      "paymentReference": null,
      "createdAt": "2026-04-04T15:30:00.000Z"
    }
  ]
}
```

| Campo | Descripcion |
|-------|-------------|
| `canonTotal` | Suma total de canon administrado (COP) |
| `leasifyFeeTotal` | Suma total del 1% Leasify (COP) |
| `estimatedCharge` | = leasifyFeeTotal (alias para claridad en UI) |
| `recordCount` | Cantidad de registros de canon |
| `history` | Ultimos 12 registros, ordenados por fecha desc |
| `history[].source` | `PSE_AUTO` (automatico al pagar via PSE) o `MANUAL` (reportado por agencia) |

**Nota:** Los registros `PSE_AUTO` se crean automaticamente cuando un cobro se marca como PAID con paymentMethod=PSE. No requiere accion del frontend.

## Flujos de Usuario para el Frontend

### Flujo 1: Landlord solicita evaluacion de un aplicante

```
1. Landlord ve un aplicante en su propiedad
2. Click "Evaluar aplicante"
3. Frontend verifica saldo: GET /agent-credits/balance
4. Si balance == 0 y tier != FLEX:
   → Mostrar modal "Compra creditos" con packs disponibles
   → POST /agent-credits/purchase con datos PSE
5. POST /evaluations/{applicationId}
6. Si 202: iniciar polling
   GET /evaluations/{applicationId}/result cada 3-5 seg
7. Cuando status == COMPLETED: mostrar resultado
8. Si 400 (creditos): mostrar "Compra creditos"
9. Si 429 (limite PRO): mostrar "Limite mensual alcanzado"
10. Si 403 (sin suscripcion): redirigir a planes
```

### Flujo 2: Agencia FLEX ve su billing

```
1. Agencia navega a seccion "Billing" en dashboard inmobiliaria
2. Frontend llama: GET /inmobiliaria/flex-billing/dashboard
3. Mostrar: canon total, fee 1% Leasify, historial
4. Filtro por ano disponible: ?year=2026
5. Para canon no-PSE: boton "Reportar canon manual"
   → Formulario con consignacion, monto, mes
   → POST /inmobiliaria/flex-billing/canon
```

### Flujo 3: Tenant ve su scoring (sin cambios, pero restringido)

```
1. Solo tenants pueden acceder a /scoring/*
2. Si un landlord intenta acceder: recibe 403
3. Landlords ahora usan /evaluations/ para ver resultados de scoring
```

### 4. AI Insights (Metricas y Actividad de Agentes)

**Roles permitidos:** Miembros de agencia (via `AgencyMemberGuard`)
**Prefijo:** `/inmobiliaria/ai`
**Header requerido:** `x-agency-id: {agencyId}`
**Permiso requerido:** `dashboard:view`

#### GET /inmobiliaria/ai/activity

Historial de actividad de los agentes AI (evaluaciones recientes de los miembros de la agencia).

**Query params:**

| Param | Tipo | Default | Max |
|-------|------|---------|-----|
| `limit` | int | 20 | 50 |

**Response 200:**
```json
{
  "activities": [
    {
      "id": "uuid",
      "agentId": "tenant-scoring",
      "agentName": "Evaluacion de Inquilino",
      "type": "execution",
      "title": "Evaluacion: Juan Perez",
      "description": "Propiedad: Apartamento Centro 301",
      "status": "success",
      "timestamp": "2026-04-04T15:30:00.000Z",
      "metadata": {
        "applicationId": "uuid",
        "durationMs": 45000,
        "result": "Completada"
      }
    }
  ],
  "source": "db"
}
```

| Campo | Valores posibles |
|-------|-----------------|
| `status` | `success` (COMPLETED), `pending` (PENDING), `failed` (FAILED) |
| `type` | `execution` (siempre, por ahora) |
| `source` | `db` (siempre — datos reales de EvaluationResult) |
| `metadata.durationMs` | Solo presente en evaluaciones completadas |

**Nota:** Retorna evaluaciones de TODOS los miembros activos de la agencia, no solo del usuario actual.

---

#### GET /inmobiliaria/ai/metrics

Metricas agregadas de los agentes AI para el dashboard de la agencia.

**Response 200:**
```json
{
  "scoring": {
    "evaluationsThisMonth": 12,
    "avgTimeMin": "3 min",
    "escalationRate": "8%",
    "accuracyRate": "92%"
  },
  "matching": {
    "suggestionsSent": 0,
    "conversionRate": "0%",
    "candidatesRedirected": 0,
    "avgCompatibility": "0%"
  },
  "summary": {
    "actionsThisWeek": 5,
    "hoursSavedThisMonth": "6h"
  }
}
```

| Campo | Descripcion |
|-------|-------------|
| `scoring.evaluationsThisMonth` | Total evaluaciones este mes (todos los miembros) |
| `scoring.avgTimeMin` | Tiempo promedio de evaluacion completada |
| `scoring.escalationRate` | % de evaluaciones fallidas |
| `scoring.accuracyRate` | % de evaluaciones completadas exitosamente |
| `matching.*` | Todo en 0 — smart matching es v1.4+ |
| `summary.actionsThisWeek` | Evaluaciones esta semana |
| `summary.hoursSavedThisMonth` | Estimado de horas ahorradas (~30min por evaluacion) |

**Nota:** Las metricas se calculan desde la DB del backend (EvaluationResult), no desde el micro de agentes. Cuando smart-matching se implemente (v1.4+), la seccion `matching` se populara con datos reales.
