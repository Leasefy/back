    1. Create `src/leases/lease-documents.controller.ts` following the exact pattern of `DocumentsController`:
       - Controller scoped to `'leases/:leaseId/documents'`
       - @ApiTags('Lease Documents'), @ApiBearerAuth()
       - Inject LeaseDocumentsService

       Endpoints:

       a) `POST /leases/:leaseId/documents` - Upload lease document
          - No @Roles decorator (both TENANT and LANDLORD can upload - service validates)
          - @UseInterceptors(FileInterceptor('file'))
          - @ApiConsumes('multipart/form-data')
          - ApiBody with file (binary) and type (LeaseDocumentType enum)
          - Params: @Param('leaseId', ParseUUIDPipe), @UploadedFile() file, @Body() dto: UploadLeaseDocumentDto, @CurrentUser() user
          - Throw BadRequestException if !file
          - Call leaseDocumentsService.upload(leaseId, user.id, dto.type, file)
          - Return 201

       b) `GET /leases/:leaseId/documents` - List lease documents
          - No @Roles (both can list - service validates)
          - Params: @Param('leaseId', ParseUUIDPipe), @CurrentUser() user
          - Call leaseDocumentsService.findByLease(leaseId, user.id)
          - Return 200 with array

       c) `GET /leases/:leaseId/documents/:id/url` - Get signed URL
          - No @Roles (both can access - service validates)
          - Params: @Param('leaseId', ParseUUIDPipe), @Param('id') documentId, @CurrentUser() user
          - Note: documentId is cuid (NOT UUID), so do NOT use ParseUUIDPipe for :id param
          - Call leaseDocumentsService.getSignedUrl(leaseId, documentId, user.id)
          - Return 200 with { url, expiresAt }

       d) `DELETE /leases/:leaseId/documents/:id` - Delete document
          - No @Roles (service validates uploader)
          - Params: @Param('leaseId', ParseUUIDPipe), @Param('id') documentId, @CurrentUser() user
          - Note: documentId is cuid (NOT UUID), so do NOT use ParseUUIDPipe for :id param
          - Call leaseDocumentsService.delete(leaseId, documentId, user.id)
          - Return { message: 'Document deleted successfully' }

       All endpoints have Swagger decorators: @ApiOperation, @ApiResponse, @ApiParam

    1. Add `getTenantDocuments(userId: string)` method to `src/users/users.service.ts`:

       This method queries 3 sources in parallel using Promise.all():

       a) ApplicationDocuments: `prisma.applicationDocument.findMany` where application.tenantId = userId
          - Include: application.id, application.property.title (for property context)
          - OrderBy: createdAt desc

       b) LeaseDocuments: `prisma.leaseDocument.findMany` where lease.tenantId = userId
          - Include: lease.id, lease.propertyAddress (for property context)
          - OrderBy: createdAt desc

       c) Contract PDFs: `prisma.contract.findMany` where tenantId = userId AND contractPdfPath is not null AND status in ['SIGNED', 'ACTIVE']
          - Select: id, contractPdfPath, createdAt, application.property.title
          - OrderBy: createdAt desc

       Map each source to a unified response shape:
       ```typescript
       interface TenantVaultDocument {
         id: string;
         name: string;
         type: 'contract' | 'receipt' | 'inventory';
         category: string;  // More specific: 'application_document', 'lease_document', 'contract_pdf'
         property: string;
         date: string;       // ISO string
         size: string;       // Human-readable (e.g., "2.4 MB")
         status: 'signed' | 'pending' | 'available';
         sourceType: 'application' | 'lease' | 'contract';
         sourceId: string;   // applicationId, leaseId, or contractId
         documentId: string; // The actual document ID for URL generation
       }
       ```

       Mapping rules for ApplicationDocument:
       - type: 'contract' for ID_DOCUMENT, 'receipt' for PAY_STUB/BANK_STATEMENT, 'inventory' for OTHER/EMPLOYMENT_LETTER
       - status: 'available'
       - property: application.property.title
       - size: format file size (bytes to human-readable)
       - name: originalName from document

       Mapping rules for LeaseDocument:
       - type: map LeaseDocumentType to frontend types:
         - CONTRACT_SIGNED, ADDENDUM -> 'contract'
         - PAYMENT_RECEIPT -> 'receipt'
         - DELIVERY_INVENTORY, RETURN_INVENTORY, PHOTO, OTHER -> 'inventory'
       - status: 'signed' for CONTRACT_SIGNED, 'available' for others
       - property: lease.propertyAddress
       - size: format fileSize to human-readable
       - name: fileName from document

       Mapping rules for Contract PDFs:
       - type: 'contract'
       - status: 'signed'
       - property: application.property.title
       - name: 'Contrato firmado'
       - size: '' (unknown, stored in Supabase)

       Helper method `formatFileSize(bytes: number): string`:
       - < 1024: `${bytes} B`
       - < 1024*1024: `${(bytes/1024).toFixed(1)} KB`
       - else: `${(bytes/(1024*1024)).toFixed(1)} MB`

       Return object:
       ```typescript
       {
         documents: TenantVaultDocument[],  // Combined, sorted by date desc
         stats: {
           total: number,
           signed: number,
           pending: number,
           available: number,
         }
       }
       ```

       NOTE: Do NOT generate signed URLs in this endpoint (too many concurrent URL generations). Instead, return the document metadata. Frontend uses the individual signed URL endpoints (existing `/applications/:id/documents/:docId/url` and new `/leases/:leaseId/documents/:id/url`) to get URLs on demand.
