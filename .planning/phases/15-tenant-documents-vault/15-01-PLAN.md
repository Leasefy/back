       ```prisma
       model LeaseDocument {
         id         String            @id @default(cuid())
         leaseId    String            @map("lease_id")
         uploadedBy String            @map("uploaded_by")
         type       LeaseDocumentType
         fileName   String            @map("file_name") @db.VarChar(255)
         filePath   String            @map("file_path") @db.VarChar(500)
         fileSize   Int               @map("file_size")
         mimeType   String            @map("mime_type") @db.VarChar(100)
         createdAt  DateTime          @default(now()) @map("created_at")

         // Relations
         lease    Lease @relation(fields: [leaseId], references: [id], onDelete: Cascade)
         uploader User  @relation("UploadedLeaseDocuments", fields: [uploadedBy], references: [id])

         @@index([leaseId])
         @@index([uploadedBy])
         @@map("lease_documents")
       }
       ```

    3. Update `src/leases/dto/index.ts` to export the new DTOs.

    4. Create `src/leases/lease-documents.service.ts` following the exact pattern of `DocumentsService`:
       - Constructor: inject PrismaService, ConfigService. Create Supabase client with service key.
       - Constants: BUCKET_NAME = 'application-documents', MAX_FILE_SIZE = 10MB, ALLOWED_MIME_TYPES (pdf, jpeg, png, webp), URL_EXPIRY_SECONDS = 3600, STORAGE_FOLDER = 'lease-documents'
       - `upload(leaseId: string, userId: string, type: LeaseDocumentType, file: Express.Multer.File)`:
         1. Find lease by ID, throw NotFoundException if not found
         2. Verify user is tenant OR landlord of the lease (`lease.tenantId === userId || lease.landlordId === userId`), throw ForbiddenException otherwise
         3. Validate file size <= MAX_FILE_SIZE
         4. Validate MIME type using magic numbers (dynamic import of file-type, same as DocumentsService.detectMimeType)
         5. Generate storage path: `lease-documents/{leaseId}/{timestamp}-{random}.{ext}`
         6. Upload to Supabase Storage (private bucket 'application-documents')
         7. Create LeaseDocument record in database
         8. If DB insert fails, delete uploaded file from storage (error handling per research pitfall 5)
         9. Return the created LeaseDocument

       - `findByLease(leaseId: string, userId: string)`:
         1. Find lease, verify access (tenant or landlord)
         2. Return all LeaseDocuments for lease, ordered by createdAt desc

       - `getSignedUrl(leaseId: string, documentId: string, userId: string)`:
         1. Find lease, verify access (tenant or landlord)
         2. Find document by ID, verify it belongs to the lease
         3. Generate signed URL with 1-hour expiry using Supabase Storage
         4. Return { url, expiresAt }

       - `delete(leaseId: string, documentId: string, userId: string)`:
         1. Find lease (no need to verify lease party - only uploader check)
         2. Find document by ID, verify it belongs to the lease
         3. Verify user is the uploader (`document.uploadedBy === userId`), throw ForbiddenException otherwise
         4. Check 24-hour deletion window: if `Date.now() - document.createdAt.getTime() > 24 * 60 * 60 * 1000`, throw BadRequestException('Documents can only be deleted within 24 hours of upload')
         5. Delete from Supabase Storage (log error but continue if storage delete fails)
         6. Delete from database

       - Private helpers: `detectMimeType(buffer)` and `getExtensionFromMime(mime)` - copy exact implementations from DocumentsService.

    Note: Do NOT import or inject DocumentsService. The existing DocumentsService is tightly coupled to ApplicationDocument and ApplicationEventService. Create an independent service that follows the same patterns but operates on LeaseDocument.
