  2. **AgentCredit model** (place after existing models, before the end of file):
  ```prisma
  /// Current credit balance for a landlord or agent user
  /// One row per user - created on first purchase, updated atomically
  model AgentCredit {
    id        String   @id @default(uuid()) @db.Uuid
    userId    String   @unique @map("user_id") @db.Uuid
    balance   Int      @default(0)

    createdAt DateTime @default(now()) @map("created_at")
    updatedAt DateTime @updatedAt @map("updated_at")

    // Relations
    user User @relation(fields: [userId], references: [id], onDelete: Cascade)

    @@index([userId])
    @@map("agent_credits")
  }
  ```
  NOTE: AgentCredit does NOT have a `transactions` back-relation. AgentCreditTransaction relates only to User (not to AgentCredit). This avoids a dual-relation on the same userId FK column which Prisma would reject.

  3. **AgentCreditTransaction model**:
  ```prisma
  /// Ledger of all credit movements (purchases and deductions)
  /// Append-only - never updated after creation
  model AgentCreditTransaction {
    id           String                      @id @default(uuid()) @db.Uuid
    userId       String                      @map("user_id") @db.Uuid
    type         AgentCreditTransactionType
    amount       Int
    balanceAfter Int                         @map("balance_after")

    // For purchases: PSE reference
    pseTransactionId String? @map("pse_transaction_id") @db.VarChar(100)
    amountPaidCop    Int?    @map("amount_paid_cop")

    // For deductions: application reference
    applicationId String? @map("application_id") @db.Uuid

    // Description for history display
    description String @db.VarChar(255)

    createdAt DateTime @default(now()) @map("created_at")

    // Relations — only to User, NOT to AgentCredit (avoids dual FK on userId)
    user User @relation(fields: [userId], references: [id])

    @@index([userId, createdAt(sort: Desc)])
    @@index([userId])
    @@map("agent_credit_transactions")
  }
  ```
  IMPORTANT: AgentCreditTransaction has a single @relation to User only. Do NOT add a relation to AgentCredit. History queries use `where: { userId }` directly — no navigation through AgentCredit needed.

  4. **User model back-relations** - add these two lines inside the User model, after the `teamMembers` relation (before the closing `@@map("users")`):
  ```prisma
  // Agent credits relations
  agentCredit              AgentCredit?
  agentCreditTransactions  AgentCreditTransaction[]
  ```
