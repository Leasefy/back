# Phase 03.2-01 Summary: Natural Language Search Parser

## Completed: 2026-02-03

## What Was Built

### NaturalSearchParserService
Created `src/properties/services/natural-search-parser.service.ts`:

- **Property Type Detection**: casa, apartamento, apto, estudio, habitacion, cuarto
- **City Recognition**: 25+ major Colombian cities (Bogota, Medellin, Cali, etc.)
- **Neighborhood Detection**: 40+ Bogota neighborhoods (Chapinero, Usaquen, Suba, etc.)
- **Bedroom Extraction**: Numbers (1-10) + Spanish words (uno, dos, tres...)
- **Bathroom Extraction**: Same pattern as bedrooms
- **Parking Detection**: parqueadero, garaje, parking
- **Amenity Keywords**: piscina, gimnasio, amoblado, terraza, etc. (15+ amenities)
- **Price Hints**: "X millones", "barato", "economico"
- **Accent Handling**: Normalizes accents for flexible matching (bogota = bogotá)
- **Remaining Text**: Unparsed words go to full-text search

### Updated Components

1. **FilterPropertiesDto**: Added `naturalQuery` parameter
2. **PropertiesService**: Integrated parser in `buildPublicWhereClause()`
3. **PropertiesModule**: Registered `NaturalSearchParserService`
4. **PropertiesController**: Updated Swagger docs with examples

## API Usage

```bash
# Natural language search
GET /properties?naturalQuery=busco%20casa%20en%20bogota%20con%202%20habitaciones

# Combined with explicit filters (explicit takes precedence)
GET /properties?naturalQuery=apartamento%20en%20chapinero&minPrice=1500000

# Complex query
GET /properties?naturalQuery=casa%20grande%20con%20piscina%20y%20parqueadero%20en%20medellin
```

## Parser Behavior

| Input | Extracted |
|-------|-----------|
| "casa en bogota" | type=HOUSE, city=Bogota |
| "2 habitaciones" | bedrooms=2 |
| "dos baños" | bathrooms=2 |
| "con parqueadero" | parkingSpaces>=1 |
| "amoblado" | amenities=[furnished] |
| "2 millones" | maxPrice=2000000 |
| "barato" | maxPrice=2000000 |
