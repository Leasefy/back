    Also update line 296 — the User model default:

    Step 2: Create the migration with `--create-only` to prevent Prisma from auto-generating destructive SQL:
    ```bash

    ```

    Step 3: Replace the generated migration SQL content with manual rename SQL:
    ```sql
    -- Rename SubscriptionPlan enum values: FREE->STARTER, BUSINESS->FLEX
    -- PostgreSQL ALTER TYPE RENAME VALUE is atomic and updates all column values in-place
    ALTER TYPE "SubscriptionPlan" RENAME VALUE 'FREE' TO 'STARTER';
    ALTER TYPE "SubscriptionPlan" RENAME VALUE 'BUSINESS' TO 'FLEX';
    ```
    IMPORTANT: If the generated SQL contains `DROP TYPE "SubscriptionPlan"` — delete it entirely and replace with the two ALTER TYPE lines above. Prisma may incorrectly generate a destructive migration.

    Step 4: Apply the migration:
    ```bash
    npx prisma migrate dev
    ```

    Step 5: Regenerate the Prisma client:
    ```bash
    npx prisma generate
    ```

    Step 6: Update the TypeScript enum in `src/common/enums/subscription-plan.enum.ts`:
    ```typescript
    /**
     * Subscription tier level enum matching Prisma schema.
     * Represents the plan tier (STARTER, PRO, FLEX).
     *
     * - STARTER: $0/mo base tier (was FREE)
     * - PRO: $149,000/mo premium tier
     * - FLEX: $0/mo, billing by canon (was BUSINESS)
     *
     * Plan limits are stored in the database via SubscriptionPlanConfig model.
     * Use the SubscriptionsService to query plan limits dynamically.
     */

      STARTER = 'STARTER',
      PRO = 'PRO',
      FLEX = 'FLEX',
    }
    ```
