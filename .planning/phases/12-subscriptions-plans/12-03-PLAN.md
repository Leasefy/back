  Create `src/subscriptions/controllers/subscription-plans.controller.ts`.

  Follow the pattern from `src/notification-templates/notification-templates.controller.ts` for admin endpoints.

  **Route prefix:** `subscription-plans`

  **Endpoints:**

  1. `GET /subscription-plans` - **Public** (no auth)
     - Decorator: `@Public()` (from `src/auth/decorators/public.decorator.ts`)
     - Query param: `planType` (optional, 'TENANT' | 'LANDLORD')
     - Calls: `subscriptionPlansService.findAll(planType)`
     - Returns: Array of SubscriptionPlanConfig
     - Swagger: @ApiOperation({ summary: 'Listar planes disponibles' })

  2. `GET /subscription-plans/:id` - **Public**
     - Decorator: `@Public()`
     - Calls: `subscriptionPlansService.findById(id)`
     - Returns: Single SubscriptionPlanConfig
     - Param validation: `@Param('id', ParseUUIDPipe)`

  3. `PATCH /subscription-plans/:id` - **Admin only**
     - Decorator: `@Roles(Role.ADMIN)`
     - Calls: `subscriptionPlansService.updatePricing(id, dto)`
     - Body: `UpdatePlanPricingDto`
     - Returns: Updated SubscriptionPlanConfig
     - Swagger: @ApiOperation({ summary: 'Actualizar precios del plan (Admin)' })

  **Swagger decorators:**
  - `@ApiTags('Planes de Suscripcion')` at class level
  - `@ApiOperation()` on each method
  - `@ApiResponse()` for success and error codes

  Create `src/subscriptions/controllers/index.ts` barrel file exporting both controllers.

  **1. Create `src/subscriptions/controllers/subscriptions.controller.ts`:**

  **Route prefix:** `subscriptions`

  All endpoints require authentication (default behavior via global guard). No @Public() decorator.

  **Endpoints:**

  1. `GET /subscriptions/me` - Get current subscription
     - Extract user from `@Req() req` (pattern: `req.user`)
     - Calls: `subscriptionsService.getActiveSubscription(user.id)`
     - Also calls: `planEnforcementService.getUsageSummary(user.id)`
     - Also calls: `subscriptionsService.getUserPlanConfig(user.id)`
     - Returns: `{ subscription: Subscription | null, usage: UsageSummary, planConfig: SubscriptionPlanConfig }`
     - This is the main "subscription dashboard" endpoint

  2. `POST /subscriptions/trial` - Start 7-day trial
     - Body: `{ planId: string }` (simple DTO with @IsUUID)
     - Calls: `subscriptionsService.startTrial(user.id, planId)`
     - Returns: Created Subscription with status=TRIAL
     - Swagger: 'Iniciar periodo de prueba de 7 dias'

  3. `POST /subscriptions/subscribe` - Subscribe to a plan
     - Body: `CreateSubscriptionDto`
     - Calls: `subscriptionsService.subscribe(user.id, dto)`
     - Returns: `{ subscription, paymentResult }`
     - Swagger: 'Suscribirse a un plan'

  4. `POST /subscriptions/cancel` - Cancel subscription
     - Body: `{ reason?: string }` (optional)
     - Calls: `subscriptionsService.cancel(user.id, reason)`
     - Returns: Updated Subscription with status=CANCELLED
     - Swagger: 'Cancelar suscripcion'

  5. `POST /subscriptions/change-plan` - Change plan mid-cycle
     - Body: `ChangePlanDto`
     - Calls: `subscriptionsService.changePlan(user.id, dto)`
     - Returns: `{ subscription, paymentResult }`
     - Swagger: 'Cambiar de plan'

  6. `GET /subscriptions/usage` - Get usage summary
     - Calls: `planEnforcementService.getUsageSummary(user.id)`
     - Returns: `{ plan, propertiesUsed, propertiesLimit, scoringViewsUsed, scoringViewsLimit }`

  **Route ordering note:** Put `GET /subscriptions/me` and `GET /subscriptions/usage` BEFORE any parameterized routes (pattern from Phase 3: static routes before `:id` routes).

  ```typescript
    // ...
  }
  ```
