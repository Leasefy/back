  <action>
  **1. `subscription-plans.service.ts` - SubscriptionPlansService:**

  Methods:
  - `findAll(planType?: PlanType): Promise<SubscriptionPlanConfig[]>` - List all active plans, optionally filtered by TENANT/LANDLORD. Used by public endpoint.
  - `findById(id: string): Promise<SubscriptionPlanConfig>` - Get single plan by ID. Throws NotFoundException.
  - `findByTypeAndTier(planType: PlanType, tier: SubscriptionPlan): Promise<SubscriptionPlanConfig>` - Look up plan config by type+tier combo. Used internally.
  - `updatePricing(id: string, dto: UpdatePlanPricingDto): Promise<SubscriptionPlanConfig>` - Admin updates plan pricing/limits. Validates plan exists, applies partial update.

  Inject: PrismaService only.

  **2. `subscriptions.service.ts` - SubscriptionsService:**

  Methods:
  - `getActiveSubscription(userId: string): Promise<Subscription | null>` - Get user's current active/trial subscription. Query: status IN (TRIAL, ACTIVE, CANCELLED) AND endDate > now. Include plan relation.
  - `getUserPlanConfig(userId: string): Promise<SubscriptionPlanConfig>` - Get the plan config for user's current subscription. If no active subscription, return the FREE plan config matching user's role (determine role from user.role / user.activeRole: if LANDLORD or activeRole=LANDLORD -> LANDLORD plans, else TENANT plans).
  - `startTrial(userId: string, planId: string): Promise<Subscription>` - Create trial subscription. Set status=TRIAL, startDate=now, endDate=now+7days, trialEndsAt=now+7days. Validate user doesn't already have an active subscription. Update user.subscriptionPlan to the plan's tier.
  - `subscribe(userId: string, dto: CreateSubscriptionDto): Promise<{ subscription: Subscription; paymentResult: any }>` - Full subscribe flow:
    1. Validate planId exists and is active
    2. Check if upgrading from trial or free
    3. If plan is FREE tier: create subscription with status=ACTIVE, no payment needed
    4. If plan is paid: process PSE mock payment using PseMockService.processPayment()
    5. If PSE returns SUCCESS: create subscription with status=ACTIVE, create SubscriptionPayment record, update user.subscriptionPlan
    6. If PSE returns FAILURE: throw BadRequestException with PSE error message
    7. If PSE returns PENDING: create subscription with status=ACTIVE (optimistic), create SubscriptionPayment with status=PENDING
    8. Cancel any existing subscription (mark EXPIRED)
    9. Calculate endDate: if MONTHLY -> startDate + 1 month, if ANNUAL -> startDate + 1 year
  - `cancel(userId: string, reason?: string): Promise<Subscription>` - Mark subscription as CANCELLED, set cancelledAt, set autoRenew=false. Subscription remains active until endDate. Throws if no active subscription.
  - `changePlan(userId: string, dto: ChangePlanDto): Promise<{ subscription: Subscription; paymentResult: any }>` - Change plan mid-cycle:
    1. Cancel current subscription (set EXPIRED)
    2. Create new subscription with new plan
    3. New cycle starts now, endDate = now + 1 month (or year)
    4. Process payment for new plan via PSE mock
    5. Update user.subscriptionPlan to new tier
  - `handleExpiredSubscriptions(): Promise<number>` - Called by cron. Find subscriptions where endDate < now AND status IN (ACTIVE, CANCELLED). Mark as EXPIRED. Downgrade user.subscriptionPlan to FREE. For landlords: hide excess properties (keep 1 published, set rest to DRAFT). Return count of expired.

  Inject: PrismaService, SubscriptionPlansService, PseMockService (import from TenantPaymentsModule - need to export it), NotificationsService.

  **IMPORTANT for PseMockService reuse:** The TenantPaymentsModule needs to export PseMockService. Add it to the TenantPaymentsModule exports if not already exported. Check `src/tenant-payments/tenant-payments.module.ts` - if PseMockService is not in exports, add it. This is a small modification to an existing file.

  **3. `plan-enforcement.service.ts` - PlanEnforcementService:**

  Methods:
  - `canPublishProperty(userId: string): Promise<{ allowed: boolean; reason?: string; currentCount: number; maxAllowed: number }>` - Check if landlord can publish another property. Get user's plan config, count current AVAILABLE+PENDING properties, compare to maxProperties (-1 = unlimited). Return result object.
  - `canViewScoring(userId: string): Promise<{ allowed: boolean; reason?: string; remainingViews: number; canMicropay: boolean }>` - Check if tenant can view scoring. Get plan config. If premium scoring or unlimited views: allowed. Otherwise check ScoringUsage for current month. If views exhausted: return allowed=false, canMicropay=true (with scoring view price).
  - `recordScoringView(userId: string): Promise<void>` - Increment ScoringUsage viewCount for current month. Upsert pattern: create if not exists, increment if exists.
  - `recordPaidScoringView(userId: string): Promise<void>` - Increment paidViewCount. Used after micropayment succeeds.
  - `getUsageSummary(userId: string): Promise<{ plan: string; propertiesUsed: number; propertiesLimit: number; scoringViewsUsed: number; scoringViewsLimit: number }>` - Summary for user dashboard.

  Inject: PrismaService, SubscriptionsService (for getUserPlanConfig).
