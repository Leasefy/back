### Task 4: Guard `AgencyPermissionGuard`

**Archivo nuevo:** `src/inmobiliaria/agency/guards/agency-permission.guard.ts`

LĂłgica:
1. Lee metadata `AGENCY_PERMISSION_KEY` del handler
2. Si no hay metadata â†’ pass (no se requiere permiso especĂ­fico)
3. Lee `agencyMemberRole` del request (ya inyectado por AgencyMemberGuard)
4. Si rol === ADMIN â†’ pass siempre
5. Busca AgencyMember en DB (por userId + agencyId) para obtener campo `permissions`
6. Si `permissions` es null â†’ usa `AGENCY_ROLE_DEFAULTS[role]`
7. Si `permissions` no es null â†’ usa custom permissions
8. Verifica que `permissions[module]?.includes(action)`
9. Si no â†’ `ForbiddenException('No tienes permiso para [action] en [module]')`

**Nota:** Este guard DEBE ejecutarse DESPUĂ‰S de `AgencyMemberGuard` que ya inyecta `agencyId` y `agencyMemberRole`.

**OptimizaciĂłn:** Para evitar doble query a DB, el guard puede hacer la query al AgencyMember una vez y cachear permissions en `request.agencyPermissions`. Considerar extender AgencyMemberGuard para que tambiĂ©n traiga `permissions` en el select.

**DecisiĂłn de implementaciĂłn:** Extender el select de `AgencyMemberGuard` para incluir `permissions`:
```typescript
// En agency-member.guard.ts, actualizar el select:
const member = await this.prisma.agencyMember.findFirst({
  where: { userId: user.id, status: AgencyMemberStatus.ACTIVE },
  select: { agencyId: true, role: true, permissions: true },
});
request.agencyPermissions = member.permissions; // Json | null
```

AsĂ­ `AgencyPermissionGuard` no necesita otra query.

### Task 5: Endpoints de permisos

**Archivo:** `src/inmobiliaria/agency/agency.controller.ts`

Agregar 2 endpoints:

```typescript
// GET /inmobiliaria/agency/members/:memberId/permissions
@Get('members/:memberId/permissions')
@UseGuards(AgencyMemberGuard)
async getMemberPermissions(
  @CurrentAgency() agency: AgencyInfo,
  @Param('memberId') memberId: string,
) {
  this.ensureAdmin(agency.memberRole as AgencyMemberRole);
  return this.service.getMemberPermissions(agency.agencyId, memberId);
}

// PUT /inmobiliaria/agency/members/:memberId/permissions
@Put('members/:memberId/permissions')
@UseGuards(AgencyMemberGuard)
async updateMemberPermissions(
  @CurrentAgency() agency: AgencyInfo,
  @Param('memberId') memberId: string,
  @Body() dto: UpdatePermissionsDto,
) {
  this.ensureAdmin(agency.memberRole as AgencyMemberRole);
  return this.service.updateMemberPermissions(agency.agencyId, memberId, dto);
}
```

**DTO:** `UpdatePermissionsDto` â€” valida que solo contenga mĂłdulos y acciones vĂˇlidos.

**Service:** `getMemberPermissions` devuelve custom permissions si existen, o los defaults del rol. `updateMemberPermissions` guarda el JSON y valida:
- No se puede modificar permisos de otro ADMIN
- Los mĂłdulos y acciones deben ser vĂˇlidos
- No se pueden asignar permisos que excedan los del rol ADMIN
