# Plan 24-03: Tests, Endpoint de Permisos Efectivos y Expiración de Invitaciones

## Goal

Validar los permisos implementados en 24-01 y 24-02 con tests de integración, agregar un endpoint unificado para consultar permisos efectivos del usuario actual, e implementar la expiración automática de invitaciones pendientes.

## Tasks

### Task 1: Endpoint `GET /users/me/permissions`

**Archivo:** `src/users/users.controller.ts`

Endpoint que devuelve los permisos efectivos del usuario actual según su contexto:

```typescript
@Get('me/permissions')
async getMyPermissions(@CurrentUser() user: User) {
  return this.usersService.getEffectivePermissions(user);
}
```

**Respuesta según tipo de usuario:**

**Para TENANT:**
```json
{
  "role": "TENANT",
  "context": "tenant",
  "permissions": null
}
```

**Para LANDLORD (owner directo):**
```json
{
  "role": "LANDLORD",
  "context": "owner",
  "teamRole": null,
  "permissions": {
    "properties": ["view", "create", "edit", "delete"],
    "candidates": ["view", "edit"],
    ...
  }
}
```

**Para LANDLORD (team member):**
```json
{
  "role": "LANDLORD",
  "context": "team_member",
  "teamRole": "manager",
  "ownerId": "uuid-del-propietario",
  "permissions": {
    "properties": ["view", "create", "edit"],
    "candidates": ["view", "edit"],
    ...
  }
}
```

**Para AGENT (agency member):**
```json
{
  "role": "AGENT",
  "context": "agency_member",
  "agencyId": "uuid-agencia",
  "agencyRole": "AGENTE",
  "permissions": {
    "dashboard": ["view"],
    "portafolio": ["view", "edit"],
    "pipeline": ["view", "create", "edit"],
    ...
  }
}
```

**Service logic:**
1. Si es TENANT → no aplica permisos granulares
2. Si es LANDLORD → buscar TeamMember con userId. Si existe → devolver permisos del rol. Si no → devolver todos (es owner)
3. Si es AGENT → buscar AgencyMember, devolver custom permissions o defaults del rol

### Task 2: Expiración automática de invitaciones

**2a. Invitaciones de agencia:**

**Archivo:** `src/notifications/scheduled/notifications-scheduler.ts` (o crear scheduler propio)

Agregar cron job que ejecute cada hora:

```typescript
@Cron(CronExpression.EVERY_HOUR)
async expireAgencyInvitations() {
  const expired = await this.prisma.agencyMember.updateMany({
    where: {
      status: AgencyMemberStatus.INVITED,
      invitationExpiresAt: { lt: new Date() },
    },
    data: { status: AgencyMemberStatus.INACTIVE },
  });
  if (expired.count > 0) {
    this.logger.log(`Expired ${expired.count} agency invitations`);
  }
}
```

**2b. Invitaciones de equipo propietario:**

```typescript
@Cron(CronExpression.EVERY_HOUR)
async expireTeamInvitations() {
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);
  const expired = await this.prisma.teamMember.updateMany({
    where: {
      status: 'pending',
      invitedAt: { lt: sevenDaysAgo },
    },
    data: { status: 'expired' },
  });
  if (expired.count > 0) {
    this.logger.log(`Expired ${expired.count} team invitations`);
  }
}
```

**Nota:** TeamMember no tiene campo `invitationExpiresAt`. Se usa `invitedAt + 7 días` como regla fija. Si se desea hacer configurable, agregar campo en migración futura.
