1. Create `src/dashboard/dto/landlord-dashboard-response.dto.ts`:
   Define response DTO with Swagger decorators:
   ```typescript
   financial: {
     monthlyIncome: number;     // Total received this month (COP)
     expectedIncome: number;    // Sum of active lease monthly rents
     collectionRate: number;    // Percentage (0-100, one decimal)
     pendingPayments: number;   // Expected minus received (COP)
     latePayments: number;      // Count of leases without payment this month
   }
   urgentActions: {
     totalUrgent: number;
     pendingApplications: number;   // SUBMITTED or UNDER_REVIEW status
     pendingSignatures: number;     // Contracts awaiting landlord signature
     pendingVisits: number;         // PENDING visit status
     endingLeases: number;          // ENDING_SOON lease status
   }
   candidates: {
     a: number;  // Risk level A count
     b: number;  // Risk level B count
     c: number;  // Risk level C count
     d: number;  // Risk level D count
   }
   ```
   Use `@ApiProperty()` with nested schema definitions. Use `!:` definite assignment.

2. Create `src/dashboard/services/landlord-dashboard.service.ts`:
   - Inject PrismaService
   - `getDashboard(landlordId: string)` method that runs three aggregations in parallel using Promise.all:
     a) `getFinancialStats(landlordId)`:
        - Get active leases: `prisma.lease.findMany({ where: { landlordId, status: { in: ['ACTIVE', 'ENDING_SOON'] } }, select: { id: true, monthlyRent: true } })`
        - Calculate expectedIncome as sum of monthlyRent values
        - Get payments received this month: Use `prisma.payment.aggregate({ where: { leaseId: { in: leaseIds }, periodMonth: currentMonth, periodYear: currentYear }, _sum: { amount: true } })`
        - Calculate collectionRate: `(received / expected) * 100`, round to 1 decimal, handle division by zero
        - Count late payments: find leaseIds that have NO payment for current month/year
        - Use `Intl.DateTimeFormat` or manual month/year extraction with `new Date()`
     b) `getUrgentActions(landlordId)`:
        - Use `getAccessiblePropertyIds(landlordId)` helper to get combined property IDs
        - Run 4 parallel count queries:
          * Applications with status IN ['SUBMITTED', 'UNDER_REVIEW'] for accessible properties
          * Contracts with status 'PENDING_LANDLORD_SIGNATURE' and landlordSignedAt IS null for landlordId
          * PropertyVisits with status 'PENDING' for accessible properties
          * Leases with status 'ENDING_SOON' for landlordId
        - Note: Use string values for Prisma status where clauses, matching the Prisma enum values (not TypeScript enums directly - cast if needed)
     c) `getCandidateRiskDistribution(propertyIds)`:
        - Get applications in reviewable statuses (SUBMITTED, UNDER_REVIEW, PREAPPROVED) for accessible properties
        - Include riskScore with `select: { level: true }`
        - Group by risk level in-memory using reduce: count A, B, C, D
        - Return `{ a, b, c, d }` (use lowercase for frontend compatibility)
   - Private `getAccessiblePropertyIds(landlordId)` helper: query Prisma directly (do NOT inject PropertyAccessService). Get owned properties via `prisma.property.findMany({ where: { landlordId }, select: { id: true } })` and agent-assigned properties via `prisma.propertyAccess.findMany({ where: { agentId: landlordId, isActive: true }, select: { propertyId: true } })`. Combine both sets of property IDs with `[...new Set([...owned, ...agentAccess])]`.

   - `@Get()` endpoint with `@CurrentUser('id') landlordId: string`
   - Return type `LandlordDashboardResponseDto`
   - Add `@ApiOperation({ summary: 'Get landlord dashboard aggregated stats' })`

3. Create `src/dashboard/services/tenant-dashboard.service.ts`:
   - Inject PrismaService
   - `getDashboard(tenantId: string)` method:
     a) Get active lease with nested data:
        ```
        prisma.lease.findFirst({
          where: { tenantId, status: { in: ['ACTIVE', 'ENDING_SOON'] } },
          include: {
            property: { select: { title: true, address: true, city: true } },
            payments: { orderBy: { paymentDate: 'desc' }, take: 3 },
            _count: { select: { payments: true } },
          },
        })
        ```
     b) If no active lease, still return useful data:
        - Count pending applications: `prisma.application.count({ where: { tenantId, status: { in: ['DRAFT', 'SUBMITTED', 'UNDER_REVIEW', 'NEEDS_INFO', 'PREAPPROVED'] } } })`
        - Get upcoming visit: `prisma.propertyVisit.findFirst({ where: { tenantId, status: { in: ['PENDING', 'ACCEPTED'] }, visitDate: { gte: new Date() } }, orderBy: { visitDate: 'asc' }, include: { property: { select: { title: true, address: true } } } })`
        - Return `{ hasActiveLease: false, pendingApplications, upcomingVisit }`
     c) If active lease exists:
        - Check current month payment: find in lease.payments array where periodMonth === currentMonth AND periodYear === currentYear
        - Calculate next payment due date: use `calculateNextPaymentDate(lease.paymentDay)` helper
        - Get upcoming visit (same as above)
        - Count pending applications (same as above)
        - Return full `TenantDashboardResponseDto`
   - Private `calculateNextPaymentDate(paymentDay: number): Date` helper:
     - Get current date
     - Create date for this month with paymentDay
     - If date has already passed this month, move to next month
     - Return the Date
