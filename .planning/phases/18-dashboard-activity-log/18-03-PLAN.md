   - `@OnEvent('payment.receiptUploaded')` handler:

       b) For landlord (userId=landlordId, actorId=tenantId): type PAYMENT_REQUEST_SUBMITTED, metadata includes { leaseId, amount, tenantName, propertyTitle: event.propertyTitle }
     - Note: Use `event.propertyTitle` and store as `propertyTitle` in metadata. Also include a made-up `propertyId` if not available in event - check the event class. If not available, omit propertyId from metadata.
   - `@OnEvent('payment.validated')` handler:
     - Receives `PaymentValidatedEvent`

     - Create TWO entries for tenant and landlord
     - Metadata: { leaseId, amount, approved: event.approved, propertyTitle: event.propertyTitle }
   - `@OnEvent('payment.disputeOpened')` handler:
     - Receives `PaymentDisputeOpenedEvent`
     - Create TWO entries:
       a) For tenant: type PAYMENT_DISPUTE_OPENED, resourceType='dispute', resourceId=disputeId
       b) For landlord: type PAYMENT_DISPUTE_OPENED
     - Metadata: { leaseId, paymentRequestId, amount, propertyTitle: event.propertyTitle }
   - All handlers wrapped in try/catch with logger.error

   - `@OnEvent('visit.requested')` handler:

     - Create TWO entries:
       a) For tenant (userId=tenantId, actorId=tenantId): type VISIT_REQUESTED, resourceType='visit', resourceId=visitId, metadata { propertyId, propertyTitle, visitDate: event.visitDate.toISOString(), startTime }
       b) For landlord (userId=landlordId, actorId=tenantId): type VISIT_REQUESTED, metadata { propertyId, propertyTitle, tenantName, visitDate, startTime }
   - `@OnEvent('visit.statusChanged')` handler:
     - Receives `VisitStatusChangedEvent` from `../../visits/events/visit-status-changed.event.js`
     - Map event.newStatus to ActivityType:
       * ACCEPTED -> VISIT_ACCEPTED
       * REJECTED -> VISIT_REJECTED
       * CANCELLED -> VISIT_CANCELLED
       * COMPLETED -> VISIT_COMPLETED
       * RESCHEDULED -> VISIT_RESCHEDULED
       * Default: return early (no activity for unknown statuses)
     - Create TWO entries:
       a) For tenant (userId=tenantId, actorId=changedByUserId): the mapped ActivityType, resourceType='visit', resourceId=visitId, metadata { propertyId, propertyTitle, newStatus, visitDate, reason: event.reason }
       b) For landlord (userId=landlordId, actorId=changedByUserId): same type, metadata { propertyId, propertyTitle, newStatus, visitDate }
     - Import VisitStatus from `../../common/enums/index.js` for the switch statement
   - All handlers wrapped in try/catch with logger.error
