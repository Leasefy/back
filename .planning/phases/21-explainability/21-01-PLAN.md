**2. Create `src/scoring/explainability/driver-formatter.service.ts`:**

`@Injectable()` class `DriverFormatterService` with one method:

```typescript
format(drivers: Driver[], signals: Signal[], result: RiskScoreResultData): DriverExplanationDto[]
```

Logic:
- For each driver, find the matching signal from the `signals` array by comparing `driver.text === signal.message`.
- If matching signal found, infer category from `signal.code`:
  - Codes containing 'RTI', 'DTI', 'BUFFER', 'INCOME', 'RENT_RATIO', 'DISPOSABLE' -> `'financial'`
  - Codes containing 'EMPLOYMENT', 'TENURE', 'EMPLOYER', 'JOB' -> `'stability'`
  - Codes containing 'REFERENCE', 'RENTAL_HISTORY', 'LANDLORD_REF' -> `'history'`
  - Codes containing 'PAYMENT_HISTORY', 'ON_TIME', 'LATE_PAYMENT', 'RETURNING', 'MONTHS_ON_PLATFORM' -> `'paymentHistory'`
  - Codes containing 'DOC_VERIFICATION', 'DOCUMENT', 'OCR' -> `'documentVerification'`
  - Default (including 'CONSISTENCY', 'MISMATCH', 'MISSING', 'DATA_') -> `'integrity'`
- If no matching signal, default to `'integrity'` category.
- Determine `icon`: positive -> `'trending_up'`, negative with integrity category -> `'warning'`, otherwise -> `'trending_down'`.
- Return array of `DriverExplanationDto` objects.
