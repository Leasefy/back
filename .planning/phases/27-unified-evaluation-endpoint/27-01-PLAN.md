  <name>Task 1: Add Prisma models and enum for evaluations</name>

  <action>
Add the following to `prisma/schema.prisma`:

1. **EvaluationStatus enum** (place near other enums):
```
enum EvaluationStatus {
  PENDING
  COMPLETED
  FAILED
}
```

2. **EvaluationResult model** (place after AgentCreditTransaction):
- `id` String @id @default(uuid()) @db.Uuid
- `applicationId` String @unique @map("application_id") @db.Uuid — one evaluation per application
- `requestedBy` String @map("requested_by") @db.Uuid — landlord/agent who triggered it
- `runId` String @map("run_id") @db.VarChar(100) — runId from micro POST /tenant-scoring
- `status` EvaluationStatus @default(PENDING)
- `result` Json? — full response from micro when complete (future-proof, no parsing)
- `error` String? @db.Text — error message if FAILED
- `createdAt` DateTime @default(now()) @map("created_at")
- `updatedAt` DateTime @updatedAt @map("updated_at")
- Relations: `application Application @relation(fields: [applicationId], references: [id], onDelete: Cascade)`, `requestedByUser User @relation("EvaluationRequester", fields: [requestedBy], references: [id])`, `transaction EvaluationTransaction?`
- Indexes: `@@index([applicationId])`, `@@index([requestedBy])`
- Map: `@@map("evaluation_results")`

3. **EvaluationTransaction model**:
- `id` String @id @default(uuid()) @db.Uuid
- `evaluationId` String @unique @map("evaluation_id") @db.Uuid
- `userId` String @map("user_id") @db.Uuid
- `tier` String @db.VarChar(20) — STARTER, PRO, or FLEX
- `amountPaidCop` Int @map("amount_paid_cop") — 0 for FLEX, 42000 STARTER, 21000 PRO
- `creditsDeducted` Int @map("credits_deducted") — 0 or 1
- `createdAt` DateTime @default(now()) @map("created_at")
- Relations: `evaluation EvaluationResult @relation(fields: [evaluationId], references: [id])`, `user User @relation(fields: [userId], references: [id])`
- Indexes: `@@index([userId])`
- Map: `@@map("evaluation_transactions")`

4. **EvaluationUsage model** (mirrors ScoringUsage pattern):
- `id` String @id @default(uuid()) @db.Uuid
- `userId` String @map("user_id") @db.Uuid
- `month` Int (1-12)
- `year` Int
- `count` Int @default(0)
- `createdAt` DateTime @default(now()) @map("created_at")
- `updatedAt` DateTime @updatedAt @map("updated_at")
- Relation: `user User @relation(fields: [userId], references: [id])`
- Unique: `@@unique([userId, month, year], name: "userId_month_year_eval")`
- Index: `@@index([userId])`
- Map: `@@map("evaluation_usage")`

5. **User model back-relations** — add these to the User model relations section:
- `evaluationResults EvaluationResult[] @relation("EvaluationRequester")`
- `evaluationTransactions EvaluationTransaction[]`
- `evaluationUsage EvaluationUsage[]`

6. **Application model back-relation** — add to Application model:
- `evaluationResult EvaluationResult?`

IMPORTANT: The `@@unique` name for EvaluationUsage must be different from ScoringUsage's to avoid conflicts. Use `userId_month_year_eval`.
