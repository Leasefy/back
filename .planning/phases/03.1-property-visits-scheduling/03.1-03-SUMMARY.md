## Decisions Made
- **Transaction for double-booking prevention:** Uses Prisma transaction to double-check slot availability at creation time, preventing race conditions
- **One active visit per property per tenant:** Tenant can only have one PENDING or ACCEPTED visit per property to prevent spam
- **Event emission pattern:** Uses EventEmitter2 to emit visit.requested event for future notification integration in Phase 13
- **VisitWithDetails interface:** Consistent response shape with property and tenant details for all visit queries
