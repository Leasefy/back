## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Role-based VisitStateMachine | Different from ApplicationStateMachine - visits require role-specific permissions (landlord accepts, tenant requests) |
| 2-hour minimum booking buffer | Prevents booking too close to now, gives landlord time to prepare |
| 30-day maximum advance booking | Limits query scope, prevents abuse, typical scheduling horizon |
| Slot duration options (15/30/45/60) | Flexibility for different property types (quick apartment vs detailed house tour) |
| On-demand slot generation | More flexible than pre-stored slots; availability rules generate slots dynamically |
| Set-based booked slots lookup | O(1) lookup for booked slots improves performance with many bookings |
