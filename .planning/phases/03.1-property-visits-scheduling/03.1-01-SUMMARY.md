## Decisions Made

| Decision | Rationale |
|----------|-----------|
| VisitStatus with 6 values | Covers full visit lifecycle: request, response, completion, cancellation, rescheduling |
| Weekly recurring availability | More flexible than pre-generated slots; landlord sets pattern, slots generated on-demand |
| Self-referential rescheduling | Tracks chain of rescheduled visits without separate table |
| Unique constraint on availability | Prevents overlapping time windows for same property/day/start time |
| Cascade delete on PropertyAvailability | When property deleted, availability config removed automatically |
| Separate tenantNotes/landlordNotes | Each party has private notes field for their comments |
