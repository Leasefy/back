<task type="auto">
  <name>Task 2: Add Prisma models and User relations</name>
  <files>
    prisma/schema.prisma
  </files>
  <action>
    Add to prisma/schema.prisma:

    1. **New enums** (after existing enums):
       ```prisma
       /// Status of a tenant payment request
       enum TenantPaymentRequestStatus {
         PENDING_VALIDATION
         APPROVED
         REJECTED
         DISPUTED
         CANCELLED
       }

       /// Status of a payment dispute
       enum PaymentDisputeStatus {
         OPEN
         UNDER_REVIEW
         RESOLVED_FAVOR_TENANT
         RESOLVED_FAVOR_LANDLORD
         CLOSED
       }

       /// Account type for bank accounts
       enum AccountType {
         AHORROS
         CORRIENTE
       }
       ```

    2. **LandlordPaymentMethod model**:
       - id, landlordId (relation to User)
       - bankName (VarChar 100), accountType (AccountType enum), accountNumber (VarChar 50), holderName (VarChar 200)
       - phoneNumber (VarChar 20, optional for Nequi/Daviplata)
       - methodType (PaymentMethod enum - reuse existing)
       - instructions (VarChar 500, optional)
       - isActive (Boolean default true)
       - createdAt, updatedAt
       - Indexes: landlordId, methodType
       - Map to: landlord_payment_methods

    3. **TenantPaymentRequest model**:
       - id, leaseId (relation to Lease), tenantId (relation to User)
       - amount (Int), paymentMethod (PaymentMethod), periodMonth, periodYear, paymentDate (Date)
       - receiptPath (VarChar 500, optional for transfers)
       - pseTransactionId, pseBankCode (optional for PSE)
       - referenceNumber (VarChar 100, optional)
       - status (TenantPaymentRequestStatus default PENDING_VALIDATION)
       - validatedAt, validatedBy (relation to User), rejectionReason (VarChar 500)
       - paymentId (unique, optional relation to Payment when approved)
       - createdAt, updatedAt
       - Indexes: leaseId, tenantId, status
       - Relation to PaymentDispute (one-to-one optional)
       - Map to: tenant_payment_requests

    4. **PaymentDispute model**:
       - id, paymentRequestId (unique relation to TenantPaymentRequest)
       - reason (Text), additionalEvidencePath (VarChar 500 optional)
       - status (PaymentDisputeStatus default OPEN)
       - resolvedAt, resolvedBy (relation to User), resolution (Text optional)
       - createdAt, updatedAt
       - Index: status
       - Map to: payment_disputes

    5. **Add relations to User model**:
       ```prisma
       landlordPaymentMethods  LandlordPaymentMethod[] @relation("LandlordPaymentMethods")
       tenantPaymentRequests   TenantPaymentRequest[]  @relation("TenantPaymentRequests")
       paymentValidations      TenantPaymentRequest[]  @relation("PaymentValidations")
       disputeResolutions      PaymentDispute[]        @relation("DisputeResolutions")
       ```

    6. **Add relation to Lease model**:
       ```prisma
       paymentRequests TenantPaymentRequest[]
       ```

    7. **Add relation to Payment model**:
       ```prisma
       paymentRequest TenantPaymentRequest?
       ```

    Use @map() for snake_case column names. Follow existing patterns in schema.
  </action>
  <verify>

    - Format: `npx prisma format`
  </verify>
