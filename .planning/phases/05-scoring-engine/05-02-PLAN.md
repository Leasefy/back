   - Export interface Signal:
     ```typescript
     export interface Signal {
       code: string;           // e.g., 'LOW_RTI', 'HIGH_DTI'
       positive: boolean;      // true = good, false = concern
       weight: number;         // points contributed
       message: string;        // human-readable explanation
     }
     ```
   - Export interface ModelResult:
     ```typescript
     export interface ModelResult {
       score: number;          // Within model's allocated range
       maxScore: number;       // Model's maximum possible score
       signals: Signal[];      // Positive/negative factors detected
     }
     ```

2. Create `src/scoring/models/financial-model.ts`:
   - Import { Injectable } from '@nestjs/common'
   - Import ScoringFeatures, ModelResult, Signal

   - Create @Injectable() FinancialModel class with:
     - private readonly MAX_SCORE = 35 (35% weight)

     - calculate(features: ScoringFeatures): ModelResult
       - Initialize score = 0, signals = []

       - Rent-to-income ratio (max 20 points):
         - RTI <= 0.25: +20 points, signal 'LOW_RTI' positive
         - RTI <= 0.30: +17 points, signal 'GOOD_RTI' positive
         - RTI <= 0.40: +10 points, signal 'MODERATE_RTI' negative
         - RTI > 0.40: +3 points, signal 'HIGH_RTI' negative

       - Debt-to-income ratio (max 10 points):
         - DTI <= 0.20: +10 points
         - DTI <= 0.35: +7 points
         - DTI <= 0.50: +4 points
         - DTI > 0.50: +0 points, signal 'HIGH_DTI' negative

       - Disposable income buffer (max 5 points):
         - const MIN_BUFFER = 500000 (COP)
         - disposable >= MIN_BUFFER * 2: +5 points
         - disposable >= MIN_BUFFER: +3 points
         - disposable < MIN_BUFFER: +0 points, signal 'LOW_BUFFER' negative

       - Return { score, maxScore: this.MAX_SCORE, signals }
  </action>
  <verify>
    - TypeScript compiles: `npx tsc --noEmit`
    - MAX_SCORE = 35
    - All thresholds have named constants or comments
  </verify>
  <done>

    FinancialModel calculates 0-35 subscore based on RTI, DTI, and disposable income.
    Signals explain each scoring factor with human-readable messages.

3. Create `src/scoring/models/integrity-engine.ts`:
   - MAX_SCORE = 25 (25% weight)
   - analyze(application: Application, features: ScoringFeatures): ModelResult
   - Start with full score (25), deduct for inconsistencies

   - Check 1 - Age vs tenure consistency (deduct 10):
     - if tenure > (age - 18) * 12: deduct, signal 'TENURE_AGE_MISMATCH'

   - Check 2 - Income vs employment type (deduct 8):
     - if UNEMPLOYED && monthlySalary > 0: deduct, signal 'INCOME_EMPLOYMENT_MISMATCH'

   - Check 3 - Student with high income (deduct 5):
     - if STUDENT && monthlyIncome > 5000000 && !additionalIncomeSource: deduct, signal 'HIGH_STUDENT_INCOME'

   - Check 4 - Unverifiable high income (deduct 5):
     - if monthlyIncome > 10000000 && !hasEmployerContact && type != SELF_EMPLOYED: deduct, signal 'UNVERIFIABLE_HIGH_INCOME'

   - Check 5 - Completeness bonus (no deduction):
     - if hasCurrentAddress && hasEmployerContact && hasLandlordReference: signal 'COMPLETE_APPLICATION' positive

   - Ensure score doesn't go negative: Math.max(0, score)
