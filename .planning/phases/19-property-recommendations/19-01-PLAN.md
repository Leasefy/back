  <action>
    1. Create `src/recommendations/scorer/match-result.interface.ts`:
       - Define `SubModelResult` interface: `{ score: number; label: string; }`
       - Define `MatchResult` interface with fields:
         - `propertyId: string`
         - `matchScore: number` (0-100)
         - `acceptanceProbability: 'alta' | 'media' | 'baja'`
         - `matchFactors: { affordability: SubModelResult; riskFit: SubModelResult; profileStrength: SubModelResult; preferences: SubModelResult; }`
         - `recommendation: string` (Spanish human-readable explanation)

    2. Create `src/recommendations/scorer/models/affordability.model.ts` (40% weight):
       - @Injectable() class `AffordabilityModel` with method `score(property, tenantProfile): SubModelResult`
       - Compute total monthly cost: `property.monthlyRent + property.adminFee`
       - Get income from `tenantProfile.applicationData?.income ?? 0`
       - If income === 0: return `{ score: 50, label: 'Sin informacion de ingresos' }`
       - Compute rentToIncomeRatio = monthlyRent / income
       - RTI <= 0.30: score 100, label "Excelente ajuste de presupuesto"
       - RTI 0.30-0.35: score 80, label "Buen ajuste de presupuesto"
       - RTI 0.35-0.40: score 60, label "Ajuste aceptable"
       - RTI > 0.40: score = Math.max(20, Math.round(60 - (RTI - 0.40) * 100)), label "Fuera de presupuesto ideal"

    3. Create `src/recommendations/scorer/models/risk-fit.model.ts` (30% weight):
       - @Injectable() class `RiskFitModel` with method `score(property, tenantProfile): SubModelResult`
       - Get riskLevel from `tenantProfile.riskData?.level`
       - If no riskLevel: return `{ score: 50, label: 'Sin evaluacion de riesgo' }`
       - Use RiskLevel enum values (imported from `@prisma/client`):
         - A: score 100, "Perfil excelente para esta propiedad"
         - B: score 85, "Buen perfil para esta propiedad"
         - C: score 60, "Perfil aceptable, puede requerir deposito adicional"
         - D: score 30, "Perfil de riesgo alto, puede requerir codeudor"
         - default: score 50, "Sin evaluacion de riesgo"

    4. Create `src/recommendations/scorer/models/profile-strength.model.ts` (15% weight):
       - @Injectable() class `ProfileStrengthModel` with method `score(property, tenantProfile): SubModelResult`
       - Evaluates how complete the tenant's profile is (more data = stronger profile):
         - Has applicationData (submitted application): +40 points
         - Has riskData (scored): +30 points
         - Has preferences set: +15 points
         - Has user firstName+lastName: +15 points
       - If total >= 85: label "Perfil completo y verificado"
       - If total >= 55: label "Perfil parcialmente completo"
       - If total >= 15: label "Perfil basico - completa tu aplicacion"
       - If total === 0: label "Sin informacion de perfil"

    5. Create `src/recommendations/scorer/models/preferences.model.ts` (15% weight):
       - @Injectable() class `PreferencesModel` with method `score(property, tenantProfile): SubModelResult`
       - Get prefs from `tenantProfile.preferences`
       - If no preferences: return `{ score: 50, label: 'Sin preferencias configuradas' }`
       - Score criteria (total 100 points):
         - City match (prefs.preferredCities includes property.city): +30 points
         - Bedroom match (prefs.preferredBedrooms === property.bedrooms): +25 points
         - Property type match (prefs.preferredPropertyTypes includes property.type): +25 points
         - Budget match (property.monthlyRent between prefs.minBudget and prefs.maxBudget): +20 points
       - Build matches array for label: "Coincide con tus preferencias: ciudad, habitaciones"
       - If no matches: label "No coincide con tus preferencias"
       - If no criteria were set (all arrays empty and nulls), return score 50

    All labels in Spanish (Colombian market). Use `import { RiskLevel } from '@prisma/client'` where needed.
    Type the tenantProfile parameter based on the return type of `UsersService.getTenantProfile()` - create a `TenantProfile` type alias or use inline typing.
  </action>
  <verify>

  </verify>

  </done>

         - baja: "Esta propiedad puede no ser la mejor opcion. {affordability.label}."
       - Return full MatchResult object

    Use the Property type from `@prisma/client` for the property parameter.
