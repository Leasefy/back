  <action>
    1. Create `src/recommendations/dto/get-recommendations.dto.ts`:
       - Class `GetRecommendationsDto` with query param validation:
         - `sort?: 'match' | 'price_asc' | 'price_desc' | 'probability'` (default 'match') - use @IsOptional(), @IsEnum()
         - `probability?: 'alta' | 'media' | 'baja'` - use @IsOptional(), @IsEnum()
         - `page?: number` (default 1) - use @IsOptional(), @Type(() => Number), @IsInt(), @Min(1)
         - `limit?: number` (default 9) - use @IsOptional(), @Type(() => Number), @IsInt(), @Min(1), @Max(50)
       - Add @ApiPropertyOptional() decorators for Swagger docs
       - Create string enums for sort and probability: `enum RecommendationSort { MATCH = 'match', PRICE_ASC = 'price_asc', PRICE_DESC = 'price_desc', PROBABILITY = 'probability' }` and `enum AcceptanceProbability { ALTA = 'alta', MEDIA = 'media', BAJA = 'baja' }`

    2. Create `src/recommendations/dto/index.ts`:
       - Export all DTOs and enums from get-recommendations.dto.ts

    3. Create `src/recommendations/recommendations.service.ts`:
       - @Injectable() class `RecommendationsService`
       - Constructor injects: PropertiesService, UsersService, RecommendationScorer
       - Method `getRecommendations(userId: string, filters: GetRecommendationsDto)`:
         a. Fetch tenant profile: `const tenant = await this.usersService.getTenantProfile(userId)`
         b. Fetch ALL available properties in one query: `await this.propertiesService.findPublic({ page: 1, limit: 1000 })` -- fetch all for in-memory scoring. NOTE: Do NOT pass status filter since `findPublic` already excludes DRAFT. Filter to AVAILABLE status in-memory after fetch if needed, or accept all non-DRAFT (AVAILABLE, PENDING, RENTED). Actually, per spec "available properties" = only AVAILABLE status. Filter in-memory: `propertiesResponse.data.filter(p => p.status === 'AVAILABLE')`
         c. Score each property in-memory: `properties.map(p => ({ property: p, match: this.scorer.score(p, tenant) }))`
         d. Filter by minimum match score >= 40
         e. Filter by probability if specified
         f. Sort by requested sort parameter:
            - 'match': descending by matchScore
            - 'price_asc': ascending by monthlyRent
            - 'price_desc': descending by monthlyRent
            - 'probability': descending by probability level (alta=3, media=2, baja=1), then by matchScore
         g. Paginate: slice(start, end) with start = (page-1)*limit
         h. Return `{ data: [...with match data merged into property], meta: { total, page, limit, totalPages, hasNext, hasPrev } }`
         - The response data items should spread the property object and add matchScore, acceptanceProbability, matchFactors, recommendation at top level

       - Method `getTopRecommendation(userId: string)`:
         a. Call getRecommendations(userId, { sort: 'match', page: 1, limit: 1 })
         b. Return data[0] ?? null

       - Method `getPropertyMatchScore(userId: string, propertyId: string)`:
         a. Fetch tenant profile
         b. Fetch property by ID: `this.propertiesService.findByIdOrThrow(propertyId)`
         c. Return `this.scorer.score(property, tenant)`

    4. Create `src/recommendations/recommendations.controller.ts`:
       - @Controller('recommendations') with @ApiTags('Recommendations')
       - @ApiBearerAuth() at class level

       - GET `/` - getRecommendations:
         - @Roles(Role.TENANT) - only tenants
         - @Query() GetRecommendationsDto
         - @CurrentUser('id') userId
         - Returns paginated recommendations
         - @ApiOperation({ summary: 'Get personalized property recommendations' })

       - GET `/top` - getTopRecommendation:
         - @Roles(Role.TENANT)
         - @CurrentUser('id') userId
         - Returns single best match or null
         - @ApiOperation({ summary: 'Get best single recommendation' })
         - IMPORTANT: Place this route BEFORE any `:id` params to prevent 'top' being parsed as ID

       - GET `/properties/:propertyId/match-score` - getPropertyMatchScore:
         - Actually, this should be at a different path. Per ROADMAP spec: `GET /properties/:id/match-score`
         - Since this is on the PropertiesController path, we have two options:
           a. Add it to RecommendationsController as GET `/property/:propertyId/match-score`
           b. Use the path `/recommendations/property/:propertyId/match-score` instead
         - Use option (a) with path `/property/:propertyId/match-score` under RecommendationsController (keeps all recommendation logic in one controller)
         - @Roles(Role.TENANT)
         - @CurrentUser('id') userId
         - @Param('propertyId') propertyId (validate UUID with ParseUUIDPipe)
         - Returns MatchResult
         - @ApiOperation({ summary: 'Get match score for a specific property' })

    Use existing decorators: @CurrentUser from src/auth/decorators/current-user.decorator.ts, @Roles from src/auth/decorators/roles.decorator.ts, Role from src/common/enums/role.enum.ts. Follow patterns from WishlistsController.
