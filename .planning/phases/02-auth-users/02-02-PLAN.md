<task type="auto">
  <name>Task 2: Create Supabase JWT strategy and guards</name>
  <files>src/auth/strategies/supabase.strategy.ts, src/auth/guards/supabase-auth.guard.ts, src/auth/guards/roles.guard.ts</files>
  <action>
1. Create src/auth/strategies/supabase.strategy.ts:
   - Extend PassportStrategy(Strategy, 'supabase')
   - Inject ConfigService and PrismaService
   - Configure JWT extraction from Bearer token
   - Use jwks-rsa with passportJwtSecret() for JWKS-based verification:
     ```typescript
     import { passportJwtSecret } from 'jwks-rsa';
     // In super() options:
     secretOrKeyProvider: passportJwtSecret({
       cache: true,
       rateLimit: true,
       jwksRequestsPerMinute: 5,
       jwksUri: configService.get('SUPABASE_JWKS_URL'),
     })
     ```
   - Set issuer to SUPABASE_URL + '/auth/v1'
   - validate() method: lookup user by payload.sub (Supabase user UUID)
   - Throw UnauthorizedException if user not found
   - Return user object (becomes request.user)

2. Create src/auth/guards/supabase-auth.guard.ts:
   - Extend AuthGuard('supabase')
   - Inject Reflector
   - Override canActivate() to check IS_PUBLIC_KEY metadata
   - If route is public, return true without auth check
   - Otherwise, call super.canActivate()

3. Create src/auth/guards/roles.guard.ts:
   - Implement CanActivate
   - Inject Reflector
   - Check ROLES_KEY metadata for required roles
   - If no roles required, return true
   - Get user from request
   - If user.role === BOTH, allow access to any role-restricted route
   - Otherwise, check if user.role is in required roles
   - Throw ForbiddenException with clear message if denied

Import User type from @prisma/client. Use .js extensions for local imports.
  </action>
  <verify>
`npm run build` succeeds.
No TypeScript errors in strategy or guards.
  </verify>
  <done>Supabase strategy validates JWTs, guards enforce auth and roles</done>
</task>
