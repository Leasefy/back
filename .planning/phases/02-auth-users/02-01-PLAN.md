<task type="auto">
  <name>Task 3: Create database trigger for user sync</name>
  <files>supabase/migrations/00001_user_sync_trigger.sql</files>
  <action>
Create directory `supabase/migrations/` if it doesn't exist.

Create SQL migration file with:

1. Function `public.handle_new_user()`:
   - SECURITY DEFINER to run with elevated permissions
   - Inserts into public.users (id, email, role, first_name, last_name)
   - Values from NEW.id, NEW.email, and raw_user_meta_data fields
   - Default role to 'TENANT' if not specified in metadata
   - Use COALESCE for optional fields

2. Trigger `on_auth_user_created`:
   - AFTER INSERT ON auth.users
   - FOR EACH ROW EXECUTE FUNCTION public.handle_new_user()

Include idempotent checks (DROP IF EXISTS before CREATE) so migration can be re-run safely.

NOTE: This migration must be run manually in Supabase SQL Editor (Dashboard > SQL Editor) since it touches the auth schema which Prisma cannot access.

Create a README.md in supabase/migrations/ explaining how to apply migrations.
  </action>
