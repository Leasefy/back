### Task 4: Verificar guards y permisos en todos los nuevos endpoints

| Endpoint | Auth | Role Check |
|----------|------|------------|
| `GET /invitations/:token` | No (Public) | Ninguno |
| `POST /invitations/:token/accept` | Sí | Ninguno (el token es la autorización) |
| `POST /invitations/:token/decline` | No (Public) | Ninguno (el token es la autorización) |
| `POST /members/:id/resend-invitation` | Sí | ADMIN de la agencia |
| `GET /onboarding-status` | Sí | Miembro activo de la agencia |

Confirmar que `AgencyMemberGuard` en `resend-invitation` verifica el rol ADMIN correctamente.

---

### Task 5: Edge cases y validaciones

Verificar comportamiento en casos borde:

1. **Token expirado:** `GET/POST /invitations/[expired-token]` → 400 con mensaje claro
2. **Token inválido:** `GET/POST /invitations/[wrong-token]` → 404
3. **Doble aceptación:** Aceptar un token ya consumido → 400 "Invitación ya usada"
4. **Usuario ya miembro:** Aceptar invitación de alguien que ya es miembro de la misma agencia → 409 Conflict
5. **Onboarding duplicado:** `POST /users/me/onboarding` dos veces como INMOBILIARIA → segundo debería devolver error o idempotencia
6. **Agencia duplicada:** Un usuario que ya tiene agencia intenta crear otra → ConflictException
