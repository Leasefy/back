# Plan 23-01: Prisma Migration + Onboarding INMOBILIARIA

## Goal

Agregar soporte para `userType: 'INMOBILIARIA'` en el onboarding de usuarios, y extender el modelo `AgencyMember` con campos para el sistema de invitaciones por token. Este plan es la base que habilita los planes 23-02 y 23-03.

---

## Tasks

### Task 1: Prisma Schema — Campos de invitación en AgencyMember

**Archivo:** `prisma/schema.prisma`

Agregar 3 campos al modelo `AgencyMember`:
```prisma
model AgencyMember {
  // ... campos existentes ...
  invitationToken     String?   @unique
  invitationExpiresAt DateTime?
  invitedEmail        String?   // Para invitar a usuarios no registrados
}
```

**Migración SQL:**
```sql
ALTER TABLE "AgencyMember"
  ADD COLUMN "invitationToken" TEXT UNIQUE,
  ADD COLUMN "invitationExpiresAt" TIMESTAMP(3),
  ADD COLUMN "invitedEmail" TEXT;
```

### Task 2: CompleteOnboardingDto — Agregar tipo INMOBILIARIA

**Archivo:** `src/users/dto/complete-onboarding.dto.ts`

Actualizar el enum `userType` para incluir `INMOBILIARIA`:
```typescript
@IsEnum(['TENANT', 'LANDLORD', 'AGENT', 'INMOBILIARIA'])
userType: 'TENANT' | 'LANDLORD' | 'AGENT' | 'INMOBILIARIA';
```

Agregar campo opcional `agency` con validación condicional (requerido si `userType === 'INMOBILIARIA'`):
```typescript
@IsOptional()
@ValidateIf(o => o.userType === 'INMOBILIARIA')
@IsNotEmpty()
@ValidateNested()
@Type(() => CreateAgencyInOnboardingDto)
agency?: CreateAgencyInOnboardingDto;
```

Crear `CreateAgencyInOnboardingDto`:
```typescript
export class CreateAgencyInOnboardingDto {
  @IsString() @IsNotEmpty() name: string;
  @IsString() @IsOptional() nit?: string;
  @IsString() @IsOptional() city?: string;
  @IsString() @IsOptional() phone?: string;
  @IsEmail()  @IsOptional() email?: string;
}
```

### Task 3: UsersService — Lógica de onboarding INMOBILIARIA

**Archivo:** `src/users/users.service.ts` (o donde esté la lógica del onboarding)

Cuando `userType === 'INMOBILIARIA'`:
1. Actualizar el usuario (igual que otros tipos)
2. Llamar a `AgencyService.createAgency(userId, agencyData)` — ya existe este método
3. El resultado incluye `{ user, agency }` en vez de solo `{ user }`

```typescript
if (dto.userType === 'INMOBILIARIA') {
  const agency = await this.agencyService.createAgency(userId, dto.agency);
  return { user: updatedUser, agency, onboardingStep: 'agency_created' };
}
```

**Inyectar `AgencyService`** en `UsersModule` / `UsersService`.

### Task 4: UsersController — Actualizar respuesta del onboarding

**Archivo:** `src/users/users.controller.ts`

El endpoint `POST /users/me/onboarding` debe devolver la agencia cuando aplique:
```typescript
// Response para INMOBILIARIA
{
  success: true,
  user: { id, email, firstName, lastName, role },
  agency: { id, name, nit, city },
  nextStep: 'complete_agency_profile'
}

// Response para otros tipos (sin cambio)
{
  success: true,
  user: { id, email, firstName, lastName, role }
}
```

### Task 5: Ejecutar migración

```bash
npx prisma migrate dev --name add_agency_member_invitation_fields
```

Verificar que:
- Los campos nuevos existen en la DB
- No hay breaking changes en registros existentes
- El cliente de Prisma se regenera correctamente
