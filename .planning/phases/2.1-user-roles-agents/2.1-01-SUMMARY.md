# Phase 2.1 Plan 01: Schema Changes Summary

**One-liner:** Replace BOTH with AGENT role, remove activeRole, add PropertyAccess and Chat models for agent delegation and application messaging.

## What Changed

### Prisma Schema Updates

1. **Role enum**: Replaced `BOTH` with `AGENT`
   - AGENT: Can manage assigned properties on behalf of landlords

2. **User model**: Removed `activeRole` field
   - No longer needed since BOTH role is removed
   - Users have a single role

3. **PropertyAccess model** (new): Agent property assignments
   - `propertyId`, `agentId`, `grantedById`
   - Unique constraint on [propertyId, agentId]
   - Cascade delete from Property

4. **ApplicationConversation model** (new): Chat for applications
   - 1:1 with Application (unique applicationId)
   - Cascade delete from Application

5. **ChatMessage model** (new): Individual messages
   - `conversationId`, `senderId`, `content`, `readAt`
   - Indexes for efficient queries

### TypeScript Code Updates

1. **Role enum**: Updated to match Prisma (AGENT instead of BOTH)

2. **RolesGuard**: Updated logic
   - Removed: `if (user.role === Role.BOTH) return true`
   - Added: `if (user.role === Role.AGENT && requiredRoles.includes(Role.LANDLORD)) return true`

3. **All controllers**: Removed `Role.BOTH` from `@Roles()` decorators
   - TENANT routes: `@Roles(Role.TENANT)` only
   - LANDLORD routes: `@Roles(Role.LANDLORD)` only (AGENT gets access via guard)

4. **UsersService/Controller**: Deprecated role switch functionality
   - `setActiveRole()` now throws ForbiddenException
   - Endpoint marked as deprecated in Swagger

5. **CompleteOnboardingDto**: Removed BOTH from UserType enum

6. **SubscriptionsService**: Fixed to not use activeRole

## Database Migration Required

The database schema change requires manual SQL execution because:
1. PostgreSQL enum values cannot be easily removed
2. The `active_role` column must be dropped before enum changes

**Migration file:** `supabase/migrations/00003_role_agent_chat.sql`

**Steps to execute:**
1. Open Supabase Dashboard > SQL Editor
2. Copy contents of migration file
3. Execute in order (the script handles dependencies)

**What the migration does:**
1. Drops `active_role` column from users table
2. Updates any BOTH users to LANDLORD
3. Adds AGENT to Role enum
4. Creates PropertyAccess table with constraints
5. Creates ApplicationConversation and ChatMessage tables
6. Adds foreign keys and indexes
