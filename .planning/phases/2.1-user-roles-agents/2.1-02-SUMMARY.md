- This means `@Roles(Role.LANDLORD)` implicitly allows AGENT users

## Deviations from Plan

### Task 3 Already Complete

**Task 3** (Remove Role.BOTH from all controller decorators) was already completed in 2.1-01. Verified with grep - no `Role.BOTH` references exist in `src/`.

### Alternative Approach for AGENT Access

The plan suggested adding `@Roles(Role.LANDLORD, Role.AGENT)` to landlord endpoints. Instead, 2.1-01 implemented a cleaner solution:

```typescript
// RolesGuard grants AGENT access to LANDLORD routes automatically
if (user.role === Role.AGENT && requiredRoles.includes(Role.LANDLORD)) {
  return true;
}
```

This is better because:
- No need to modify every controller decorator
- Centralized logic in one place
- Easier to maintain
