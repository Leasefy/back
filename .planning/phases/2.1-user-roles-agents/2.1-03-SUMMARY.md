# Phase 2.1 Plan 03: PropertyAccess Service Summary

**One-liner:** Created PropertyAccessService with canAccessProperty, ensurePropertyAccess, assignAgent, removeAgent methods and updated 6 services to use it for agent authorization.

## What Changed

### New Files Created

1. **PropertyAccessService** (`src/property-access/property-access.service.ts`)
   - `canAccessProperty(userId, propertyId)`: Returns true if user is landlord OR has active PropertyAccess
   - `ensurePropertyAccess(userId, propertyId)`: Throws ForbiddenException if no access
   - `assignAgent(landlordId, propertyId, agentEmail)`: Assigns agent by email
   - `removeAgent(landlordId, propertyId, agentId)`: Soft-deletes agent access
   - `getAgentsForProperty(propertyId)`: Lists active agents
   - `getAccessibleProperties(agentId)`: Lists properties agent can manage
   - `getLandlordForManagedProperty(agentId, propertyId)`: Gets landlord info

2. **PropertyAccessController** (`src/property-access/property-access.controller.ts`)
   - `POST /properties/:propertyId/agents` - Assign agent by email
   - `GET /properties/:propertyId/agents` - List agents for property
   - `DELETE /properties/:propertyId/agents/:agentId` - Remove agent

3. **AgentsController** (`src/agents/agents.controller.ts`)
   - `GET /agents/me/properties` - Get properties assigned to current agent
   - `GET /agents/me/properties/:propertyId/landlord` - Get landlord info for managed property

## Key Design Decisions

### 1. Authorization Pattern
All services now use a consistent pattern:
```typescript
await this.propertyAccessService.ensurePropertyAccess(userId, propertyId);
```

This replaces the previous direct checks:
```typescript
if (property.landlordId !== landlordId) {
  throw new ForbiddenException('...');
}
```

### 2. Agent Role in State Machines
When agents perform actions (accept visit, activate contract), they act with `LANDLORD` role for state machine validation purposes. This is because:
- State machines expect `TENANT` or `LANDLORD`
- Agents act on behalf of landlords
- The RolesGuard already grants AGENT access to LANDLORD routes

### 3. Legal Record Integrity
When creating contracts, the `landlordId` field stores the actual property owner's ID, not the acting agent's ID:
```typescript
landlordId: application.property.landlordId, // actual owner, not acting user
```

This ensures legal documents reflect the true parties.
