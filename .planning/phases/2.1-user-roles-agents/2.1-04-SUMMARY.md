# Phase 2.1 Plan 04: Application Chat System Summary

**Chat service for tenant-landlord/agent communication using Supabase Realtime**

## One-liner

ChatModule with conversation lifecycle hooks (create on submit, delete on reject/withdraw) and 4 REST endpoints for messaging.

## What Was Built

### ChatService (src/chat/chat.service.ts)
Core service with methods:
- `getOrCreateConversation(applicationId)` - idempotent creation
- `getConversationWithMessages(applicationId, userId)` - fetch with sender info
- `sendMessage(applicationId, senderId, content)` - create message
- `markAsRead(applicationId, userId)` - update readAt for unread messages
- `getUnreadCount(applicationId, userId)` - count unread
- `deleteConversation(applicationId)` - cascade delete

Access verification:
- Tenant (applicant) always has access
- Landlord (property owner) always has access
- Agent with active PropertyAccess has access

### ChatController (src/chat/chat.controller.ts)
4 endpoints under `/applications/:applicationId/chat`:
| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Get conversation with messages |
| POST | `/messages` | Send a message |
| PATCH | `/read` | Mark messages as read |
| GET | `/unread` | Get unread message count |

### Application Lifecycle Integration
- **Submit**: Creates conversation via `chatService.getOrCreateConversation()`
- **Withdraw**: Deletes conversation via `chatService.deleteConversation()`
- **Reject**: Deletes conversation via `chatService.deleteConversation()`

## Key Implementation Details

### Supabase Realtime Pattern
Backend writes messages to `chat_messages` table. Frontend subscribes to Supabase Realtime channel for that table. Supabase handles WebSocket broadcast - no custom WebSocket server needed.

### Message Access Control
```typescript
// Tenant always has access
if (application.tenantId === userId) return;

// Landlord always has access
if (application.property.landlordId === userId) return;

// Agent needs PropertyAccess
const hasAccess = await this.propertyAccessService.canAccessProperty(userId, propertyId);
if (hasAccess) return;

throw new ForbiddenException('No tienes acceso a esta conversacion');
```

### Read Receipt Logic
Only marks messages from OTHER users as read:
```typescript
await this.prisma.chatMessage.updateMany({
  where: {
    conversationId: conversation.id,
    senderId: { not: userId },  // Not your own messages
    readAt: null,
  },
  data: { readAt: new Date() },
});
```
